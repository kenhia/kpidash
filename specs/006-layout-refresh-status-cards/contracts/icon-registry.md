# Contract — Dashboard-Owned Icon Registry

**Sprint**: 006  
**Owner**: dashboard (compiled in; not loaded from Redis)  
**Consumers**: Service Status Cards (this sprint); future widgets may opt in.  

---

## Purpose

Provide a stable mapping from a small integer (the only thing clients ship in a payload) to a nerd-font glyph rendered by the dashboard. This decouples client publishers from the specific font, codepoint, and rendering details (FR-023, FR-024).

## Storage format (source of truth)

A static C array in `src/icons.c`, declared in `src/icons.h`:

```c
typedef struct {
    int         index;        /* stable, append-only */
    uint32_t    codepoint;    /* nerd-font codepoint */
    const char *label;        /* short human label, mockup/debug only */
} icon_entry_t;

extern const icon_entry_t ICON_REGISTRY[];
extern const size_t       ICON_REGISTRY_COUNT;
```

Example initial entries (illustrative; final set chosen during mockup work):

```c
const icon_entry_t ICON_REGISTRY[] = {
    {  0, 0xF300, "computer" },
    {  1, 0xF301, "server"   },
    {  2, 0xF302, "database" },
    {  3, 0xF303, "queue"    },
    /* ... append-only ... */
};
```

## Lookup API

```c
/* Returns a NUL-terminated UTF-8 string for the glyph, or NULL if index is unknown. */
const char *icons_lookup(int index);

/* Returns the codepoint, or 0 if index is unknown. (Mainly for the mockup catalogue.) */
uint32_t icons_codepoint(int index);
```

## Indexing rules

- Indices are **append-only**. Never reuse, never renumber.
- An index gap (e.g. retiring index 5) is permitted; `icons_lookup(5)` simply returns NULL.
- Recommended convention: keep `index == position_in_ICON_REGISTRY` while the array is dense; once an entry is deleted, switch to a sparse search keyed on `entry.index`.

## Rendering contract

- The icon font is `lv_font_icons_56` (see `research.md` R1), generated by `fonts/generate.sh`. The Service Status Card uses an `lv_label` styled with this font and sets its text to the UTF-8 string returned by `icons_lookup()`.
- If `icons_lookup()` returns NULL (unknown index from a client payload, per FR-026), the card MUST render with no icon — the label is either hidden or its text set to `""` — and the rest of the card MUST be unaffected.

## Source font

`fonts/ttf/SymbolsNerdFont-Regular.ttf` (already vendored). The curated codepoint subset for the icon font is the union of codepoints listed in `ICON_REGISTRY`. The codepoint list passed to `lv_font_conv` MUST be kept in sync with the registry (script logic detailed in implementation).

## Versioning

The registry is part of the dashboard binary. Adding new glyphs requires a dashboard rebuild and redeploy. Clients always send integer indices, never codepoints — so dashboard upgrades cannot break existing publishers.

## Out of scope this sprint

- Dynamic registry loaded from a resource file or Redis.
- Per-card colour tinting of the icon (the colour scheme used in mockup work is sprint-internal until SC-011 sign-off).
- A discovery API for clients to enumerate available indices.
